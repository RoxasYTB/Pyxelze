Contributing to Pyxelze

Thanks for your interest in contributing! A few basic rules:

- Open an issue for bugs and feature requests.
- Create a branch per topic: `git checkout -b feature/your-feature`.
- Keep changes small and focused; include tests when possible.
- Run formatting: `dotnet format` and linting before submitting.

Pull request checklist:

- [ ] CI passes
- [ ] No secrets in the diff
- [ ] Update `CHANGELOG.md` (if relevant)
- [ ] Add tests for bug fixes or new features

Please read `SECURITY.md` for vulnerability reporting instructions.
